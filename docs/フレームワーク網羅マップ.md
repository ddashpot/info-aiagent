# フレームワーク網羅マップ（脆弱性カタログのカバレッジ）

> **目的**: 脆弱性カタログ（`ai_security_catalog.json`・34件）が、準拠を掲げる各フレームワークの公式項目を **どこまで網羅し、どこが未参照か** を一覧化する。
>
> **方針（厳守）**: カタログは**変更していない**。各フレームワークの項目は**公式ソースから転記**（末尾に出典）。カタログ側の対応は **カタログ内の実際の参照ID（`atlas` / `nist_rmf` / `category`）から機械抽出**したものであり、**推測でマッピングを足していない**。出典に基づく当方補足は「（補足）」と明示する。
>
> **更新（カタログ v1.4.0）**: 「34だけではない」とのご指摘を受け、モデル層攻撃の大分類 `ml` と脆弱性4件（`ML-01`〜`ML-04`）を追加し、**計38件**へ拡張。準拠フレームワークに **CISA ZTMM v2.0 / CSA AICM v1.0 / CSA AI Security Maturity Model** を追加（計8）。これにより参照する MITRE ATLAS 技術-ID は **19 → 28** に増加。詳細は §8。
>
> 内訳: **38件 = OWASP LLM Top10(10) ＋ OWASP Agentic Threats(15) の完全網羅 ＋ モデル層 ml(4) ＋ 独自追加(infra4 / mcp2 / multimodal3 = 9)**。残る網羅余地（ATLAS全技術・NIST/ISO統制・CISA/CSA成熟度の項目化）は §7 を参照。

## 6. CISA Zero Trust Maturity Model v2.0（成熟度評価軸）

> ご指示により**成熟度評価**の観点として追加。CISA ZTMM は脆弱性カタログ（攻撃面の列挙）とは別軸の **「ゼロトラスト成熟度の自己評価」フレーム**。エージェントのID・権限・可観測性・データ保護の成熟度を測る軸として併用できる。

**5本柱 × 4成熟度ステージ ＋ 3横断能力**

| 本柱(Pillar) | 概要 |
|---|---|
| Identity | 利用者・サービス・**エージェント等の非人間ID**の認証・検証 |
| Devices | デバイス／ワークロード実行環境の健全性 |
| Networks | ネットワーク分離・暗号化・トラフィック制御 |
| Applications & Workloads | アプリ／ワークロードの保護・認可・実行制御 |
| Data | データの分類・保護・アクセス制御・来歴 |

| 成熟度ステージ | 内容 |
|---|---|
| Traditional | 手動・静的・サイロ化 |
| Initial | 一部自動化・初期の最小権限 |
| Advanced | 全社的なポリシー・自動化の拡大 |
| Optimal | 動的・完全自動・継続的検証 |

| 横断能力(Cross-Cutting) | 内容 |
|---|---|
| Visibility & Analytics | 可観測性・分析 |
| Automation & Orchestration | 自動化・オーケストレーション |
| Governance | ガバナンス |

**（補足・要レビュー）当カタログ項目との関連の目安**（※カタログにCISA参照は無く、以下は出典に基づく当方の対応案）:
- Identity 柱 ↔ `INF-01`（ID・認証基盤）, `ASI-T09`（なりすまし）, `ASI-T03`（権限侵害）, `LLM06`（過剰エージェンシー）
- Applications & Workloads 柱 ↔ `ASI-T02`/`ASI-T11`（ツール悪用・RCE）, `MCP-01`/`MCP-02`
- Data 柱 ↔ `LLM02`（機微情報漏えい）, `ASI-T01`（メモリ汚染）, `LLM08`
- Networks 柱 ↔ `INF-04`（Egress制御）
- Governance 横断 ↔ `INF-03`（可観測性）, `ASI-T08`（否認・追跡不能）

## 6bis. CSA（Cloud Security Alliance）AI フレームワーク

> ご指示により追加。CSA は AI のセキュリティ統制と成熟度の体系を提供する（CISA とは別組織）。カタログ `meta.frameworks` に登録済み。

### CSA AI Controls Matrix (AICM) v1.0
- **18セキュリティドメイン / 243統制目標**。CCM（Cloud Controls Matrix）を基盤に、信頼できるAIの開発・運用統制を体系化。
- ISO/IEC 42001・NIST AI 600-1・BSI AIC4 にマッピング。AI-CAIQ（評価質問票）と STAR for AI 認証への経路を提供。
- 当カタログとの統制単位での対応付けは今後の作業（推測で付与しない）。

### CSA AI Security Maturity Model (AISMM)
- AICM に整合した**成熟度指標**。AIセキュリティ実践の有効性・拡張性を評価する軸。
- CISA ZTMM（ゼロトラスト成熟度）と併用し、ID/権限/データ/可観測性の成熟度評価に使える。

## 7. 残課題（網羅の余地・「34だけではない」の中身）

1. **MITRE ATLAS**: 全約60技術のうちカタログ参照は28技術-ID（v1.4.0）。残る未参照技術（偵察・探索・横展開・攻撃ステージング系の大半）を、項目として扱うか／既存脆弱性へのベクトルとして紐づけるかは要判断。
2. **NIST AI RMF**: 参照は24サブカテゴリ。全72サブカテゴリの対応一覧化が可能。
3. **ISO/IEC 42001**: 目的レベル対応のみ。38統制の個別対応一覧化が可能。
4. **CISA ZTMM**: カタログ内に参照なし。成熟度評価としてカタログ／自己評価ツールへ組み込む場合は対応付け（上記補足）のレビューが必要。

> いずれも **カタログへの項目追加・対応付けの確定は未実施**（勝手に追加しない方針）。本マップは「公式項目 × 既存参照」の事実関係のみを示す。
