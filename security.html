01使い方は3ステップ

インストール不要。ブラウザだけで始められ、入力はこの端末に保存されます。

1

自己採点する

34の脆弱性ごとに、対策が「実装済／一部／未対応」かを選ぶだけ。総合スコアが出ます。

2

弱点を把握する

未対応で深刻な項目が、直すべき順番（残存リスク順）で自動的に並びます。

3

深掘りする（任意）

レッドチーム計画や自動スキャナ（aiva）で、実際に攻撃を試して確かめます。

03スキャナを使う（CLI・上級者向け）

既存アプリを能動検査します。まずは mock でデモ（ネットワーク・APIキー不要）。

⚠ 能動検査は対象所有者の許可が前提の防御的用途です。mock 以外は --authorize を明示してください。
bash
# aiva コマンドをインストール
pip install -e tools/aiva

# デモ検査（ネットワーク不要）
aiva scan --target mock --format md,html,sarif

# 網羅性の確認（設定監査込み）
aiva coverage --arch examples/architecture.example.json

# 自社アプリ＋外部ツール統合（所有/許可している対象のみ）
aiva scan --config your.json --authorize --with-tools mcp-scan --format sarif

04このスイートの考え方

一度きりのチェックではなく、更新して使い続け、繰り返し再検査するための設計です。

体系的

もれなく整理

脆弱性を「どこが攻撃されるか × どう失敗するか」の2軸で重複なく整理しています。

既知＋未知

未知も探す

フレームワーク対応の既知パターンに加え、入力を変異させて想定外の挙動も探します。

手薄を可視化

空白がわかる

どのリスクの対策が薄いかを aiva coverage で確認できます。

継続運用

繰り返し再検査

新しい脅威をカタログに取り込み、GitHub Action で継続的に再検査できます。
