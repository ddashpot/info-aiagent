スキャナを使う（CLI）

能動検査は対象所有者の許可が前提の防御的用途です。mock以外は --authorize を明示してください。
pip install -e tools/aiva            # aiva コマンド
aiva scan --target mock --format md,html,sarif   # デモ（ネットワーク不要）
aiva coverage --arch examples/architecture.example.json   # 網羅性（設定監査込み）
aiva scan --config your.json --authorize --with-tools mcp-scan --format sarif   # 自社アプリ＋外部ツール統合

このスイートについて

  • MECE設計：脆弱性は surface(10)×failure_mode(7) の直交軸、検査は 生成戦略×判定オラクル の2軸。
  • 既知＋未知：OWASP/ATLAS対応プローブに加え、変異エンジン・回帰的深掘り・挙動異常検知で未知を炙り出す。
  • 網羅性：能動検査＋外部ツール（25+）＋設定監査で全脆弱性をカバー。aiva coverage で空白を可視化。
  • 回帰運用：脅威インテイク（収集→反映→回帰）とGitHub Actionで継続的に再検査。
